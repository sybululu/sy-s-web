### 3.5 模型加载工作流（启动时一次性执行）

> **2026-04-18 更新** — 采用 `CustomBertMoeModel` 自定义类方案
>
> **问题根因**：训练代码中分类层命名为 `self.fc`（`nn.Linear(768, 11)`），
> 而 HuggingFace 标准库 `AutoModelForSequenceClassification` 硬编码要求分类层名为 `classifier`。
>
> **解决方案**：在 `app.py` 中完全复现训练时的模型类结构，使 checkpoint 键名自然匹配，
> 无需任何字符串 replace 操作。这是比键名重映射更专业、更可靠的做法。
>
> **后续修复（同日）**：
> - `forward` 增加 `**kwargs` 兼容 tokenizer 多参数传入
> - `max_length` 从 512 改为 150，对齐训练时 `pad_size` 配置

```
应用启动 (uvicorn app:app --port 7860)
    │
    ├── RoBERTa 分类模型 ───────────────────────────────────┐
    │                                                         │
    │  ① 定义 CustomBertMoeModel 类（与训练代码结构一致）     │
    │                                                         │
    │     class CustomBertMoeModel(nn.Module):                │
    │         self.bert = BertModel.from_pretrained(          │
    │             "hfl/chinese-roberta-wwm-ext"               │
    │         )   # 基础 BERT 编码器                         │
    │         self.fc = nn.Linear(768, 11)  # ← 与训练一致！  │
    │                                                         │
    │     def forward(self, input_ids, attention_mask=None, **kwargs):│
    │         # **kwargs 兼容 tokenizer 传入的 token_type_ids 等额外参数  │
    │         outputs = self.bert(input_ids, attention_mask)           │
    │         return self.fc(outputs.pooler_output)                    │
    │                                                         │
    │  ② AutoTokenizer.from_pretrained(                      │
    │       "hfl/chinese-roberta-wwm-ext"                   │
    │     )                                                  │
    │                                                         │
    │  ③ hf_hub_download(                                   │
    │       repo_id="sybululu/bert-moe",                     │
    │       filename="multi_classification_bertmoe.ckpt"     │
    │     ) → ckpt_path                                     │
    │                                                         │
    │  ④ torch.load(ckpt_path, map_location="cpu",           │
    │              weights_only=True)                        │
    │     state_dict = {                                     │
    │       "model.bert.embeddings.*": Tensor,               │
    │       "model.bert.encoder.*":   Tensor,               │
    │       "model.fc.weight":        Tensor([11,768]), ✅   │
    │       "model.fc.bias":          Tensor([11])     ✅   │
    │     }                                                  │
    │                                                         │
    │  ⑤ 键名清理（仅需一步）：                              │
    │     cleaned = {k.replace("model.", ""): v              │
    │               for k,v in state_dict.items()}           │
    │     → bert.*, fc.*   （无需 fc→classifier 重映射！）    │
    │                                                         │
    │  ⑥ model.load_state_dict(cleaned, strict=True)         │
    │     → 所有键精确匹配，strict=True 通过                 │
    │     → 无 missing / unexpected                          │
    │                                                         │
    │  ⑦ model.eval() → 关闭 dropout                         │
    │                                                         │
    │   "✅ RoBERTa 分类模型加载完成（CustomBertMoeModel + fc）"│
    │                                                         │
    └─────────────────────────────────────────────────────────┘
```

#### 3.5.1 为什么自定义类优于 replace 方案

| 对比维度 | replace 方案（旧） | CustomBertMoeModel（新） |
|---------|------------------|------------------------|
| **原理** | 字符串替换伪造键名 | 类结构天然匹配 |
| **维护性** | 训练代码改了命名就要跟着改 | 跟随训练代码自动一致 |
| **strict 模式** | 必须 `strict=False` | 可以 `strict=True`（更安全） |
| **输出格式** | 标准 `SequenceClassifierOutput` | 原始 tensor（需适配 predict 函数） |
| **风险点** | 替换可能误伤其他键 | 无 |

#### 3.5.2 roberta_predict 函数适配

由于 `CustomBertMoeModel.forward()` 返回原始 logits tensor（非标准库的 `.logits` 对象），predict 函数做了兼容处理：

```python
inputs = tokenizer_roberta(sentence, return_tensors="pt", truncation=True, max_length=150)

#                                                         与训练时 pad_size=150 对齐

outputs = model_roberta(**inputs)
# 兼容自定义模型（返回 tensor）和标准库（返回 .logits 属性）

#### 3.5.3 生产环境适配要点（2026-04-18 补充）

| 问题 | 现象 | 根因 | 修复 |
|------|------|------|------|
| **TypeError** | `forward() got an unexpected keyword argument 'token_type_ids'` | 训练代码 `forward(self, x)` 只接收位置参数，但 transformers 的 `model(**inputs)` 会传入 `token_type_ids` 等额外关键字参数 | forward 签名增加 `**kwargs` |
| **语义特征偏移** | 生产环境分类准确率与训练时不一致 | API 中 `max_length=512`，但训练 Config 设定 `pad_size=150`，截断策略不同导致输入分布差异 | 统一为 `max_length=150` |
| **403 Forbidden** | `/api/v1/rectify` 调用 HF Inference API 返回 403 | `HF_TOKEN` 未勾选 "Make calls to the serverless Inference API" 权限 | ✅ 已解决：改用 GitHub Models API（`GITHUB_TOKEN`），HF_TOKEN 仅负责下载模型权重 |

## 七、错误处理与容错

### 7.1 前后端错误映射

| 后端错误 | HTTP状态码 | 前端处理 |

```
后端服务
    │

    │
    ├─ 模型加载失败 → 返回静态法律依据，不阻断分析

    │
    └─ 全部RAG失败 → 使用 violation_mapping.json 静态映射
