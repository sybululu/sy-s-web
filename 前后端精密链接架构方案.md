## 二、前后端接口精密对接

### 2.1 接口映射表

| 前端操作 | 前端API调用 | 后端端点 | 认证 | 数据流向 |
|---------|------------|----------|------|---------|
| 用户登录 | `api.login(email, password)` | `POST /api/v1/auth/login` | 否 | 前端→后端→返回token |
| 用户注册 | `api.register(email, password, name)` | `POST /api/v1/auth/register` | 否 | 前端→后端 |
| 提交分析 | `api.analyze(text, source_type)` | `POST /api/v1/analyze` | JWT | 前端→后端AI分析→返回结果 |
| 生成整改 | `api.rectify(snippet, violation_type)` | `POST /api/v1/rectify` | JWT | 前端→后端RAG生成→返回建议 |
| 上传文件 | `api.uploadFile(file)` | `POST /api/v1/upload` | JWT | 前端→后端解析→返回文本 |
| 抓取URL | `api.fetchUrl(url)` | `POST /api/v1/fetch-url` | JWT | 前端→后端爬取→返回文本 |
| 获取列表 | `api.getProjects()` | `GET /api/v1/projects` | JWT | 前端→后端→返回项目数组 |
| 获取详情 | `api.getProject(id)` | `GET /api/v1/projects/{id}` | JWT | 前端→后端→返回完整violations |
| 导出报告 | `api.exportReport(id)` | `GET /api/v1/export/{id}` | JWT | 前端→后端→浏览器下载 |

### 3.2 12类违规定义（论文表3-1）

| 序号 | ID | 违规指示符名称 | 权重 | 合规维度 |
|------|-----|---------------|------|---------|
| 1 | I1 | 过度收集敏感数据 | 0.15 | 数据收集 |
| 2 | I2 | 未说明收集目的 | 0.12 | 数据收集 |
| 3 | I3 | 未获得明示同意 | 0.15 | 数据收集 |
| 4 | I4 | 收集范围超出服务需求 | 0.10 | 数据收集 |
| 5 | I5 | 未明确第三方共享范围 | 0.08 | 数据共享 |
| 6 | I6 | 未获得单独共享授权 | 0.12 | 数据共享 |
| 7 | I7 | 未明确共享数据用途 | 0.08 | 数据共享 |
| 8 | I8 | 未明确留存期限 | 0.05 | 数据留存 |
| 9 | I9 | 未说明数据销毁机制 | 0.05 | 数据留存 |
| 10 | I10 | 未明确用户权利范围 | 0.05 | 用户权利保障 |
| 11 | I11 | 未提供便捷权利行使途径 | 0.03 | 用户权利保障 |
| 12 | I12 | 未明确权利响应时限 | 0.02 | 用户权利保障 |

**权重总和 = 1.00**

### 3.3 评分计算流程

```
输入: 违规类型ID列表 ["I1", "I3", "I7"]
         │
         ▼
┌─────────────────────────────────────────┐
│         Scorer.calculate_score           │
│                                         │
│   公式: S = 100 - Σ(wi × vi × 100)      │
│                                         │
│   初始值: S = 100                       │
│   I1: S = 100 - (0.15 × 1 × 100) = 85  │
│   I3: S = 85 - (0.15 × 1 × 100) = 70   │
│   I7: S = 70 - (0.08 × 1 × 100) = 62   │
│                                         │
│   最终评分 = 62.0                       │
└─────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│         Scorer.get_risk_level            │
│                                         │
│   62.0 → "中等风险"                      │
│   (40 ≤ 62 < 70)                        │
│                                         │
│   风险等级划分:                          │
│   - 低风险: S ≥ 70                      │
│   - 中等风险: 40 ≤ S < 70               │
│   - 高风险: S < 40                      │
└─────────────────────────────────────────┘
```


## 四、RAG技术路线

### 4.1 RAG完整流程

```
用户输入的违规片段
        │
        ▼
┌──────────────────┐
│ DeepDoc Parser   │ ← 文档结构化、语义分块
│ (文档解析)        │   - 支持TXT/PDF/HTML
└──────────────────┘   - 保持语义完整性
        │              - 按段落+标点切分
        ▼
┌──────────────────┐
│   BERT 向量化     │ ← multi_classification_bertmoe
│ (vectorizer)     │   - 768维语义向量
└──────────────────┘   - L2归一化
        │
        ├──────────────────┐
        ▼                  ▼
┌──────────────┐    ┌──────────────┐
│ 向量召回      │    │ BM25召回      │
│ (FAISS)       │    │ (全文检索)    │
│ Top-20       │    │ Top-20       │
└──────────────┘    └──────────────┘
        │                  │
        └────────┬─────────┘
                 ▼
┌──────────────────────────┐
│    RRF 融合              │
│ (倒数排名融合)            │
│ Top-10 综合结果           │
└──────────────────────────┘
        │
        ▼
┌──────────────────┐
│ Cross-Encoder    │ ← BERT-MOE重排序
│ (精排)            │   Top-5 高相关
└──────────────────┘
        │
        ▼
┌──────────────────┐
│ mT5 Generator    │ ← rewrite_mT5_small.ckpt
│                  │   temperature=0.1
│                  │   max_length=512
└──────────────────┘
        │
        ▼
   整改建议文本 + 法律依据
```

## 七、错误处理与容错

### 7.1 前后端错误映射

| 后端错误 | HTTP状态码 | 前端处理 |
|---------|-----------|---------|
| JWT过期/无效 | 401 | 清除token，跳转登录页 |
| 用户不存在 | 401 | 提示"邮箱或密码错误" |
| 项目不存在 | 404 | 提示"项目不存在" |
| 文本长度不足 | 422 | 提示"文本长度不足10字符" |
| 模型未加载 | 503 | 提示"系统繁忙，请稍后重试" |
| RAG检索失败 | 500 | 使用静态法律依据降级 |

### 7.2 降级策略

```
后端服务
    │
    ├─ 模型正常 → 完整RAG分析
    │
    ├─ 模型加载失败 → 返回静态法律依据，不阻断分析
    │
    ├─ Milvus不可用 → 仅用BM25召回
    │
    └─ 全部RAG失败 → 使用 violation_mapping.json 静态映射
```
